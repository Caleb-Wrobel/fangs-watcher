# Publishing — how an impl becomes a deployable image

This repo **builds and publishes images**; the [fangs](https://github.com/) config repo **composes**
them onto the embassy box. The split is the whole architecture: this repo knows nothing about the
host, TLS, the pod, or the fleet — it produces a versioned image per impl and stops there.

## Where images live

**GHCR**, one image per impl, named for the impl:

```
ghcr.io/caleb-wrobel/fangs-watcher/watcher-scala
ghcr.io/caleb-wrobel/fangs-watcher/watcher-go      # when it lands
```

The repo is **public**, which buys two free things that shape everything below: GitHub's free
runners, and **anonymous image pulls**. The embassy box pulls its image at deploy with *no registry
credential and no WireGuard* — which is exactly the charter's runtime-independence rule (an image
pulled at deploy sits locally afterward; the watcher keeps watching with the fleet dark).

## Architecture: amd64 only

The box is the Oracle **x86 E2.1.Micro**, so images are `linux/amd64` and CI runs on the default
`ubuntu-24.04` (x86) runners — arch-matched, no multi-arch matrix, and no native-image
cross-compile problem to ever solve. (morel, the fleet's amd64 host, is the arch-matched fallback
builder if runners are ever wanted; auxin arm64 cannot build for an x86 box.)

## JVM first; native-image is a later chapter

Impls publish a **JVM image** (`eclipse-temurin` JRE + one assembly jar). GraalVM native-image is a
deliberate later CI tier (release-tag only, `native-image-agent` metadata generated by the test
run) — not a child-#3 concern. The runtime is the same either way: `podman run`, `WATCHER_*` env,
one process.

## What publishes, and when

| Trigger | Builds | Smoke-tests | Publishes |
|---|---|---|---|
| `pull_request` | ✅ | ✅ | ❌ — never |
| push to `main` | ✅ | ✅ | `:edge`, `:sha-<12>` |
| release `scala-vX.Y.Z` | ✅ | ✅ | `:X.Y.Z`, `:latest` (latest skipped on prereleases) |

**Per-impl versioning.** N impls of one contract means each tongue versions on its own line, so a
Scala fix never bumps Go's number. Releases are tagged **`scala-vX.Y.Z`** (Go will be `go-vX.Y.Z`);
the image tag drops the prefix (`scala-v0.1.0` → `watcher-scala:0.1.0`). A release whose tag isn't
`scala-v*` triggers a cheap skipped job here, nothing more.

**Start at `0.1.0`, not `1.0.0`.** The watcher passes the contract and publishes, but hasn't watched
anything in production (no limen cutover, no TLS). `0.x` = works, surface may still move; reserve
`1.0.0` for after the production cutover proves it as the real dead-man's switch.

**fangs pins an immutable tag** — a specific `:X.Y.Z` (or a `@sha256:` digest) in the Quadlet, never
`:edge`/`:latest`. Cutting a release is therefore the deploy signal: tag `scala-vX.Y.Z` here, bump
the pin there. The Containerfile's base images are digest-pinned, so a rebuilt release tag is
byte-reproducible.

## The publish gate — nothing ships that failed SPEC.md

Two gates, both green before any push:

1. **Contract suite, dev path** — `./smoke/smoke.py <impl>`, the executable definition of "passes
   the contract" (down/recovery/restart/fault rules). Proves the *code*.
2. **Boot-and-ping, the built image** — run the container, assert `/healthz` 200 and SPEC's
   `404`/`200`/`404`-on-GET route contract. Proves the *artifact* (base image, entrypoint, bind).

The shared smoke harness has no "point me at a container" mode yet, which is why the image check is
a route subset rather than the full suite. Teaching `smoke.py` an `--image` mode collapses these two
gates into one — a good future cleanup, not a blocker.

## Secrets

None are ever built into an image or committed. `WATCHER_TOKEN` and `WATCHER_DISCORD_WEBHOOK` are
runtime environment only (SPEC § Configuration); in production they come from fangs vault into the
Quadlet. CI needs no secret beyond the automatic `GITHUB_TOKEN` (scope `packages: write`) to push to
GHCR.

## Adding an impl

Same as adding a language anywhere else here — a directory — plus two files that live *with* it:
a `Containerfile` and a `.github/workflows/watcher-<lang>.yml` (copy the Scala pair, swap the build
stage and the paths filter). The harness, this policy, and fangs's composition layer are untouched.

## Build any impl locally

```sh
podman build -t watcher-scala:dev scala
podman run --rm \
  -e WATCHER_TOKEN=dev -e WATCHER_DISCORD_WEBHOOK=http://127.0.0.1:9099/hook \
  -e WATCHER_SUBJECT=limen -e WATCHER_BIND=0.0.0.0 -p 127.0.0.1:8080:8080 \
  watcher-scala:dev
```
